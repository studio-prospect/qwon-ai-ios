## Rename boundaries

Rename only active QWON implementation surfaces.

| Area | Rename target | Notes |
| --- | --- | --- |
| Xcode target / scheme | `PREXUS` -> `QWON` | Split into its own PR because scheme/CI/script churn is high |
| Swift module / product references | `PREXUS` -> `QWON` where generated by Xcode target naming | Keep runtime behavior unchanged |
| iOS source paths | `app/ios/PREXUS` -> `app/ios/QWON` only after target/scheme rename is green | Avoid path rename in the same PR as behavior changes |
| Test targets | `PREXUSTests` / `PREXUSUITests` -> QWON names after app target is stable | Preserve test contents unless import names require updates |
| Scripts/docs active QWON commands | Update commands that point at the active target/scheme | Historical PREXUS command snippets stay historical |

Do **not** rename:

| Preserve | Reason |
| --- | --- |
| PREXUS historical docs and frozen ledgers | Audit trail |
| PREXUS ASC Apple ID `6775110218` and Bundle ID `jp.studio-prospect.prexus.ios` | Historical product line |
| PREXUS TestFlight tags / evidence folders | Historical release lineage |
| `prexus-local-mvp.gguf` filename | Model placement contract; change separately only if product approves a model migration |
| Closed PR titles / commit messages / tag names | Immutable history |

## Review checklist

Codex should block a Phase 4 PR when any of these are true:

| Blocker | Why |
| --- | --- |
| Historical PREXUS ledgers are rewritten | Breaks audit trail |
| Bundle ID changes away from `jp.studio-prospect.qwon.ios` | Breaks ASC / signing line |
| PREXUS ASC `6775110218` is touched for QWON | Product-line mixup |
| Behavior changes are mixed into rename PRs | Hard to review regressions |
| Committed project includes llama-linked generated state unintentionally | Breaks clean-checkout simulator tests |
| Test plan omits regenerated Xcode project after iOS/project changes | Stale project risk |
| Build/test command still references a removed active scheme | CI / developer workflow break |

Non-blocking if documented:

| Acceptable | Condition |
| --- | --- |
| Some `PREXUS` strings remain | They are historical, model filenames, closed release records, or explicitly deferred |
| Project path stays `PREXUS.xcodeproj` in PR 4B | The PR only owns target/scheme rename and defers path rename |
| No TestFlight upload | Phase 4 is internal naming unless product requests a binary respin |

---

## Stop conditions

Stop Phase 4 and return to planning if:

- Xcode generation cannot produce a no-llama clean project.
- Simulator tests fail for reasons unrelated to stale names.
- Signing or Bundle ID values drift.
- Rename churn touches historical PREXUS docs beyond links to this plan.
- A real TestFlight release blocker appears during the rename series.
