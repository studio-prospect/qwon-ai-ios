# Pitch

## One-Line Pitch

PREXUS is a local-first AI runtime for iPhone that thinks nearby, routes intelligently, and calls the cloud only when necessary.

## Short Pitch

ほとんどの AI アプリは、すべてをクラウドへ送り、毎回大きなコンテキストを投げて処理している。PREXUS は逆をやる。iPhone 上のローカル LLM を中核に、会話、OCR、音声、画像理解、文脈圧縮、メモリ処理をできる限り端末側で行い、本当に必要なときだけ高性能クラウドモデルへエスカレーションする。

その結果、低遅延、低コスト、高プライバシー、オフライン性を両立した「携帯型 AI 実行基盤」を実現する。

## Problem

現行の AI 体験には、以下の構造的な問題がある。

- クラウド前提で遅い
- 毎回トークンを大量消費する
- 機密情報を外に出しやすい
- オフラインで価値が落ちる
- モデルの使い分けが難しい

ユーザーは賢く AI を使いたいのに、現実には「どのモデルに、どの文脈を、どのタイミングで渡すか」を自分で毎回判断させられている。

## Solution

PREXUS は、ローカル LLM を単なる小型チャットモデルではなく、オーケストレータとして使う。

PREXUS が担うこと:

- 入力の intent 解析
- タスク複雑度の判断
- 機密性の評価
- 文脈圧縮
- ローカルメモリの選択的想起
- 最適な推論先の自動選択

簡単な仕事はローカルで終える。難しい仕事だけを、必要最小限のコンテキストでクラウドへ送る。

## Why Now

- iPhone の Neural Engine と端末性能が十分に強くなってきた
- 小型ローカルモデルの実用性が上がっている
- API コストとプライバシー意識が無視できなくなっている
- マルチモデル運用の複雑さが一般化し始めている

今は、クラウド AI を呼び出す UI ではなく、「近くで考えるランタイム」を作るタイミングに入っている。

## Differentiation

PREXUS は以下で差別化する。

- ローカル LLM 中心の設計
- セマンティックルーティング
- コンテキスト圧縮によるトークン削減
- iPhone センサー活用のマルチモーダル設計
- パーソナルメモリ
- クラウド依存最小化

## MVP

最初のリリースで成立させる要素:

- iPhone 上のローカル会話
- OpenAI / Anthropic 接続
- 自動モデル振り分け
- コンテキスト圧縮
- OCR と基本 Vision 入力
- API キー設定

## Long-Term Vision

PREXUS はチャットアプリを作るのではない。将来的には、個人の近くで継続的に動作するモバイル AI OS の知能レイヤを作る。

目的は、AI を「呼び出すもの」から「そばで動き続けるもの」に変えることにある。
