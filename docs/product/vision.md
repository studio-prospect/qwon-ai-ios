# Vision

## QWONとは何か

QWON は、iPhone 上で常時使えるローカルファーストの AI ランタイムである。

目指しているのは、単なるチャットアプリではない。ユーザーの近くで動作し、必要に応じて考え、見て、聞いて、記憶し、必要時のみクラウド知能を呼び出す「携帯型知能レイヤ」である。

## なぜ作るのか

現在の主要な AI 体験には共通する問題がある。

- ほぼすべてをクラウドに送る
- 毎回大きな文脈を渡す
- コストが高い
- 待ち時間が長い
- オフラインで弱い
- プライバシー境界が曖昧
- モデル選択が利用者に押し付けられがち

QWON は、この前提を変える。

まずローカルで理解し、分類し、圧縮し、必要なら記憶し、それでも足りないときだけクラウドへエスカレーションする。

## ビジョン

QWON の長期ビジョンは、「クラウド AI を呼び出すアプリ」ではなく、「個人の近くで継続的に動作するモバイル AI OS の核」を作ることにある。

このビジョンは次の原則に基づく。

- ローカルファースト
- 低遅延
- 高プライバシー
- 低トークン消費
- セマンティックルーティング
- マルチモーダル理解
- ユーザー固有メモリ

## プロダクトの核

QWON の価値は、会話 UI そのものではなく、裏側のランタイムにある。

中核となる能力:

- ローカル LLM を使った intent 解析
- タスクごとの最適実行先の自動選択
- クラウド送信前のコンテキスト圧縮
- OCR / 音声 / 画像入力のローカル前処理
- 個人向け長期メモリ
- 必要時のみのクラウド連携

## 理想的な体験

ユーザーはモデル名やプロンプト構造を毎回意識しない。

期待する体験は次のようなもの:

- すぐ反応する
- オフラインでも最低限使える
- 見せたもの、話したこと、過去の文脈を必要十分に理解する
- 機密な内容はむやみに外へ出さない
- 高度な処理だけを賢く外部モデルへ渡す

## 誰のためのプロダクトか

最初の対象は、AI を積極的に活用するヘビーユーザーである。

- ソフトウェア開発者
- AI パワーユーザー
- 研究者
- クリエイター
- API やモデルの特性を理解している利用者

ライトユーザー向けの汎用アシスタント体験を最初から狙わない。まずは要求水準の高い利用者にとって、明確に優れた runtime を作る。

## 差別化

QWON の差別化ポイントは次の通り。

- ローカル LLM が補助ではなく中核
- クラウド依存を最小化
- モデル自動選択を前提化
- 文脈圧縮でトークン浪費を抑制
- iPhone のカメラ、マイク、センサーを活かしたマルチモーダル設計
- 個人専用の知能レイヤとして育つ構造

## UIデザインの方向性

QWON の UI は Apple プラットフォームらしさを強く持ってよいが、見た目の主役は常に content ではなく runtime の使いやすさであるべきである。

そのため、Liquid Glass の採用方針は全面適用ではなく、操作レイヤーへの選択的適用を基本とする。

- navigation と control surfaces には採用しうる
- 長文コンテンツ面や高密度情報面には過剰適用しない

詳細は `docs/design/liquid_glass_adoption_strategy.md` を参照。

アプリアイコンの方向性は `docs/design/prexus_app_icon_strategy.md` を参照。

## 成功の定義

QWON が成功している状態は、次の条件を満たしていること。

- iPhone 単体で十分に有用なローカル会話ができる
- クラウドを使う場面が賢く限定されている
- 応答速度がクラウド依存アプリより体感的に速い
- コンテキスト圧縮の効果が数値でも分かる
- OCR や音声入力が実用に足る
- 利用を重ねるほど、ユーザー固有の文脈理解が強くなる

## 長期的な到達点

将来的には QWON を、会話アプリから一段上の「知能実行基盤」に進化させる。

想定している拡張:

- エージェント的なタスク実行
- iOS ショートカットや端末機能との連携
- LAN 内モデルとの接続
- Mac との連携
- MCP 対応
- 常時知能層としての動作
